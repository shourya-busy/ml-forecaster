Why can't I add new algorithms from this page? Adding a model means executing new Python inside the worker — that's an arbitrary-code-execution surface and doesn't belong behind a web form. The {{ total_registered }} algorithms above span every major forecasting family. If you genuinely need another, add a file under src/forecaster/models/, decorate the class with @register("name"), and rebuild — it appears here automatically. See docs/adding-an-algorithm.md.

Outlier filtering (preprocessing)

Optionally drop anomalous points from the training history before fitting models, so a deployment spike or monitoring blip doesn't pull the forecast off-trend. Uses scikit-learn's Isolation Forest on sliding-window features.
Note: this is preprocessing, not a forecaster. Isolation Forest itself does not produce future-value predictions; that's why it's here and not in the Algorithm library above.
